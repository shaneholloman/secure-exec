## Codebase Patterns
- Use `pnpm turbo run check-types` for TypeScript typecheck (not `typecheck`)
- Rust crate lives at `crates/v8-runtime/`; binary name is `secure-exec-v8`
- `cargo check` in `crates/v8-runtime/` validates Rust compilation
- rusty_v8 crate is imported as `v8` (e.g., `v8 = "130"` in Cargo.toml)
- IPC enums use `#[serde(tag = "type")]` (internally tagged) for JS interop
- Use `serde_bytes` for `Vec<u8>` fields to get MessagePack bin format
- Use `rmp_serde::to_vec_named` (not `to_vec`) for named-field serialization
- Framing functions: `write_message(&mut writer, &msg)` and `read_message(&mut reader)` in `ipc.rs`
- Max IPC message size is 64MB (`MAX_MESSAGE_SIZE` constant in `ipc.rs`)
- V8 tests MUST run with `--test-threads=1` to avoid SIGSEGV from parallel V8 operations
- `signal-hook::flag::register` clears the AtomicBool to `false` on signal receipt
- `v8::Context::new(scope, Default::default())` — v130 requires a `ContextOptions` second arg
- Use `v8::Global<v8::Context>` to persist context handles across HandleScope lifetimes
- Consolidate V8 tests into a single `#[test]` fn to avoid inter-test isolate state issues
- V8 platform MUST be initialized on the main thread before spawning session threads (SIGSEGV otherwise)
- Session tests use `#[cfg(not(test))]` to skip V8 isolate creation; V8 lifecycle tested in `execution::tests`
- ALL V8 tests live in `execution::tests::v8_consolidated_tests` — do NOT add new `#[test]` fns that create V8 isolates
- Use `rmpv::Value` for dynamic V8 ↔ MessagePack conversion; `rmpv::encode::write_value` for encoding, `rmpv::decode::read_value` for decoding
- Bridge functions use `v8::FunctionTemplate::builder(callback).data(external).build(scope)` to attach per-function data via `v8::External`
- Split `scope.throw_exception(v8::Exception::error(scope, msg))` into two statements to avoid double mutable borrow
- `BridgeCallContext` uses `Mutex<Box<dyn Write + Send>>` / `Mutex<Box<dyn Read + Send>>` for testable IPC abstraction
- `BridgeFnStore` must be kept alive alongside the V8 context — it owns the heap allocations pointed to by v8::External
- Use `obj.set_integrity_level(scope, v8::IntegrityLevel::Frozen)` for Object.freeze()
- Use `obj.define_own_property(scope, key, val, READ_ONLY | DONT_DELETE)` for non-writable, non-configurable properties
- Use `isolate.set_allow_wasm_code_generation_callback(fn)` to disable WASM — callback returns false
- Use `scope.get_current_context()` to get the context from within a ContextScope
- `session_id` is `String` in IPC types (not u32/u128) for JS interop; use hex nonces for 128-bit security
- Session concurrency is configurable via `SECURE_EXEC_V8_MAX_SESSIONS` env var (default: available CPUs)
- Use `std::thread::available_parallelism()` instead of `num_cpus` crate for CPU count
- `v8::PromiseResolver::new(scope)` + `resolver.get_promise(scope)` to create promise-returning functions
- Use `RefCell<HashMap<u32, v8::Global<v8::PromiseResolver>>>` for single-threaded pending promise storage
- `scope.perform_microtask_checkpoint()` flushes .then() handlers after promise resolution
- `async_send` and `sync_call` share the `next_call_id` counter — no cross-convention call_id collisions
- `TimeoutGuard::new(ms, isolate_handle, abort_tx)` spawns a timer thread; `cancel()` or `Drop` joins it; `timed_out()` checks if it fired
- Use `crossbeam_channel::select!` + abort channel pattern to unblock both `ChannelMessageReader` and `run_event_loop` on timeout
- `cpu_time_limit_ms` is set on `CreateSession` and passed to the session thread; the `TimeoutGuard` is created per-`Execute`
- `execute_script(scope, bridge_code, user_code)` in `execution.rs` compiles+runs CJS code and returns `(i32, Option<ExecutionError>)`
- `extract_error_info(scope, exception)` extracts structured error info from V8 exception values via property access (constructor.name, message, stack, code)
- `extract_process_exit_code(scope, exception)` returns `Option<i32>` — detects ProcessExitError via `_isProcessExit: true` sentinel, extracts numeric exit code
- `exception_to_result(scope, exception)` combines both: returns `(exit_code, ExecutionError)` with correct exit code for ProcessExitError
- Use `tc.exception()` then pass result + tc to helper — Local phantom lifetime doesn't block mutable scope reuse
- `v8::script_compiler::compile_module(scope, &mut source)` — takes `&mut Source`, not owned `Source`
- `module.get_identity_hash()` returns `NonZero<i32>` (not plain `i32`) — use as HashMap key directly
- Use `v8::ScriptOrigin::new(scope, name.into(), 0, 0, false, -1, None, false, false, true, None)` for ESM modules (is_module=true)
- Module resolve callback uses thread-local `RefCell<Option<ModuleResolveState>>` — brief borrows only, never hold across V8/IPC calls (recursive callbacks would double-borrow)
- Use `unsafe { v8::CallbackScope::new(context) }` to get a scope inside V8 callbacks (ResolveModuleCallback)
- `execute_module(scope, bridge_ctx, bridge_code, user_code, file_path)` returns `(i32, Option<Vec<u8>>, Option<ExecutionError>)` — exports are msgpack bytes
- `module.get_status() == v8::ModuleStatus::Errored` + `module.get_exception()` for post-evaluate error checking
- `v8::ValueSerializer::new(scope, Box::new(delegate))` + `write_header()` + `write_value(context, val)` + `release()` for V8-native serialization
- `v8::ValueDeserializer::new(scope, Box::new(delegate), data)` + `read_header(context)` + `read_value(context)` for V8-native deserialization
- Must `use v8::ValueSerializerHelper` and `use v8::ValueDeserializerHelper` to access methods on the serializer/deserializer structs
- V8 serialization handles circular references, Date, Map, Set, RegExp natively — Error objects become plain objects (lose constructor name)
- `Context::new()` on a snapshot-restored isolate clones the snapshot's default context with all state — use this instead of `get_current_context()` for fresh context per execution
- Snapshot defaults: use `READ_ONLY` only (no `DONT_DELETE`) so `inject_globals_from_payload` can redefine per-session; `DONT_DELETE` makes non-configurable
